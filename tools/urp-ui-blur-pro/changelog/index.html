v1.1.0 2026-07-20
Added
Update Mode — blur no longer has to re-render every single frame. Set it to run at a capped rate, or freeze it entirely for panels that don't need to keep redrawing, like pause menus and modals. Skipped frames cost nothing extra.
Quick-create menu items — GameObject → UI → Blur Image and GameObject → 3D Object → Blur Object (also available from the Hierarchy + button) now create an already-wired panel or 3D object with a default profile assigned, in one click.
Components no longer start blank — adding a BlurImage or BlurObject via Add Component now automatically assigns a default blur profile.
Performance controls on every inspector — the Update Mode settings are now visible directly on BlurObject and standalone Blur Profile assets, not only when editing a profile through a BlurImage.
Smarter Update Mode guidance — the inspector now estimates roughly how many frames your chosen rate actually saves at your target frame rate, making it easier to pick a sensible value.
Changed
The algorithm cost indicator now looks and behaves the same everywhere it appears, across every inspector.
Fixed
The active-user count for a BlurObject could occasionally show 0 in the Inspector even when the object was enabled and correctly set up, if the Game view hadn't rendered it recently.
v1.0.0 2026-06-25
Added
Blur Algorithms
Six blur algorithms: Dual Kawase, Gaussian Separable, Tent/Hex, Poisson Disk, Custom Kawase, and Radial — all built natively on Unity 6's URP RenderGraph API.
Blur strength scales automatically with screen resolution, so it looks consistent from 1080p to 4K without manual tuning per platform.
Components
BlurImage — attach blur to any UI panel, with its own independent appearance settings: tint, brightness, desaturation, vignette, and glass distortion.
Built-in fade in / fade out for blur strength — smooth by default, and works correctly even while the game is paused.
BlurObject — the same blur system for 3D meshes, like windows and glass panels, with realistic light refraction.
Runtime Control
Blur intensity can be read and changed at runtime (0–1), for fading or scaling blur strength without touching the underlying settings.
Update Mode lets you choose how often blur re-renders — every frame, a capped rate, or frozen entirely for static panels.
Shared Profiles
Up to 16 independent blur configurations (Blur Profiles) can run at once, and any number of panels can share one profile at zero extra GPU cost.
Editor Tooling
A Setup Wizard automatically adds the blur feature to your URP renderers on first import, and can be reopened any time from the Tools menu.
Profile management is automatic — profiles stay in sync across your project and clean up after themselves when no longer used.
Inspectors for all three components show live previews, an at-a-glance performance cost, and, in Play mode, direct controls for testing.
Included
Ready-to-use materials for both UI and 3D use — no extra setup required.
An interactive demo scene comparing all six blur methods side by side.
Fixed
Blur appearing stretched horizontally in some modes at certain screen resolutions.
Blur darkening unexpectedly at high quality settings.
Blur panels drifting slightly out of alignment at 4K resolution.
A rare case where one panel's blur could appear inside another panel.
A render-order conflict when multiple panels used the same blur algorithm.
A brief grey flash on panels while editing in the Unity Editor.
A panel with no profile assigned could occasionally show another panel's blur.
A minor memory leak on 3D blur objects over long play sessions.
Blur occasionally showing incorrect content immediately after entering Play mode.
A dark fringe appearing around panels using strong glass distortion — the blurred area now automatically extends to cover it.
